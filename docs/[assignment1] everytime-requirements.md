# 에브리타임 게시글 관련 기능 명세

## 1. 게시글 목록 조회

게시판 메인 화면에서 게시글 목록을 보여주는 기능이다.

### 필요한 데이터
- **게시글 제목** (`title`)
- **내용 미리보기** (`content`) — 전체 내용을 저장하되, 목록에서는 일부만 노출 (클라이언트 처리)
- **댓글 수** (`commentCount`)
- **공감 수** (`likeCount`)
- **작성 시각** (`createdAt`) — "14분 전" 등 상대 시간 표현은 클라이언트 처리
- **글쓴이** (`author`)

### 응답 형태
- 단건이 아닌 **리스트** 형태로 응답해야 한다.

---

## 2. 게시글 상세 조회

목록에서 게시글을 탭하면 열리는 상세 화면이다. 목록보다 더 많은 정보가 필요하다.

### 필요한 데이터
- **게시글 식별자** (`id`) — 특정 게시글 하나를 조회하기 위한 고유 식별자
- **제목** (`title`)
- **전체 내용** (`content`) — 목록과 달리 전체 내용을 보여줌
- **작성자 정보** (`author`)
- **작성 시각** (`createdAt`)
- **공감 수** (`likeCount`)
- **댓글 수** (`commentCount`)
- **댓글 목록** — 별도의 댓글 데이터 (3차 세미나에서 다룰 예정)

### 핵심 포인트
- "몇 번 게시글인지" 특정해야 하므로 게시글마다 고유한 **식별자(id)** 가 반드시 필요하다.

---

## 3. 게시글 생성

목록 화면 하단의 "글 쓰기" FAB 버튼을 통해 진입하는 글 쓰기 화면이다.

### 서버가 받아야 하는 데이터
- **제목** (`title`) — 필수값, 최대 50자 제한
- **본문** (`content`)
- **익명 여부** (`isAnonymous`) — 익명 체크박스

### 유효성 검증
- **제목 필수값 검증**: 제목이 비어있으면 등록 불가 (완료 버튼 비활성)
- **제목 길이 검증**: 최대 50자 초과 시 서버에서 거부
  - 클라이언트에서 50자 제한을 걸어도 API 직접 호출로 우회 가능하므로, **서버에서도 반드시 유효성 검증**을 해야 한다.
